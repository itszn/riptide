R.I.P.Tide (CVE-2018-4192) is a race condition vulnerability in Webkit's Riptide garbage collector

Details

FAQ

What is R.I.P.Tide?

R.I.P.Tide is a vulnerability in Webkit's Riptide garbage collector that leads to arbitrary code execution in the Safari browser

What is Riptide?

Riptide is a concurrent garbage collector built into the Webkit framework, and thus in Safari and other browsers that use webkit. Riptide will collect javascript garbage (like the language itself) and throw it out in the background while the rest of javascript runs. This has given webkit a 5% speed boost when running your 50mb of javascript frameworks. It also allows you to UAF any javascript object the nation state actors want, all at the same time!

Am I vulnerable?

R.I.P.Tide was fixed in Safari 11.1.1, shipped in MacOS version 10.13.5 and iOS 11.4

If your macbook or iPhone is behind this version, then yes anyone can run arbitrary code from your browser

Is this vulnerability serious?

R.I.P.Tide can be triggered simply by visiting any website with javascript enabled. Luckily you do not have javascript enabled for untrusted sites. This is not a perfect protection, but you are a lot safer than all those other scrubs who actually like to enjoy the internet

If you are still vulnerable, that means you have not updated your iPhone/macbook. Several root exploits have also been released for these versions, so it is trivial for any one to fully exploit your device as it stands.

How can I detect exploitation?

If a website downloads javascript from a server, and tries to have your browser run it, its probably malicious, and you should report it to the cyber police asap

Can my antivirus detect or block this attack?

The exploit involves taking a javascript array and reversing it. That's it. Good luck writing a signature for that doesn't falsely detect the rest of the random javascript frameworks (Although maybe the world would be a better place if they were all blocked)
